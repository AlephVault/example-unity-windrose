Detailed Description

Object strategies are the counterpart of World.ObjectsManagementStrategies.ObjectsManagementStrategy, and will reside in the same object holding an ObjectStrategyHolder.

Quite often they hold state instead of logic, and notify the counterpart management strategy in the map by invoking PropertyWasUpdated(string, object, object).
