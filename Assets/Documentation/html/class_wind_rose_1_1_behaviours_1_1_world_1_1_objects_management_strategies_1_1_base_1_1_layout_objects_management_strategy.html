Detailed Description

Layout management strategies involve the ability of tiles to tell whether nothing should move through them: movement will be forbidden on tiles that are marked as "blocking".

Its counterpart is Objects.Strategies.Base.LayoutObjectStrategy.

See also
ObjectsManagementStrategy, Objects.Strategies.Base.LayoutObjectStrategy

override void WindRose.Behaviours.World.ObjectsManagementStrategies.Base.LayoutObjectsManagementStrategy.ComputeCellData ( uint  x,
uint  y 
)
virtual

Computing the per-cell data involves keeping the nearest block flag. This means: the last flag, among all tilemaps, in the x and y position, will determine whether the cell is blocked or not. Flag can only be extracted from a tile, if the tile is a BundledTile and contains a LayoutTileStrategy.

Parameters
xThe X position to compute
yThe Y position to compute

If no tile is BundledTile with the expected LayoutTileStrategy, the computed value is false.
