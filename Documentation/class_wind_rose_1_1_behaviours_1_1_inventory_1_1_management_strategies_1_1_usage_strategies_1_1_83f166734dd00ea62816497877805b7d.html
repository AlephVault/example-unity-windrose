Detailed Description

Usage strategies try consuming or using certain items. They should have a chained behaviour (i.e. depend among them), and ultimately a base usage strategy would try consuming quantities of the stack being used.

They will interact with stacks having compatible usage (state) strategies, and will run the usage behaviour in coroutines, so interactions with UI and even asynchronous server connection are feasible.

Uses a certain stack, which would be present in the inventory. The stack will have a compatible strategy to be used. This method may dispatch calls to DoUse(Stack, object) defined in dependencies (and related strategies in the stack will be attended there).

This method is the key. It should:

  • Use a stack only in terms of the counterpart-expected behaviour. If such counterpart behaviour is missing, then it should cry.
  • Delegation is done through yield return anotherComponent.DoUse(stack, argument).
  • If you are at DoUse is because you are aware that the stack has certain main usage strategy and it will have their dependencies as well. Delegating will imply that this inventory usage strategy will find the compatible dependencies for such stack's dependencies. Ensure the dependencies are properly set in both stack usage strategies and inventory usage strategy.

There is an optional argument to customize the usage type. It may be null, so when that happens you must be prepared to implement a default usage.
