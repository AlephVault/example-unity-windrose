Detailed Description

A sub-renderer is, basically, a view than can be connected to an InventorySimpleBagRenderingManagementStrategy. It is not just a way to render items, but also a way to interact with them by -e.g.- pagination: different sub renderers may show different pages, but they will render the same underlying items.

◆ Refresh()

virtual void WindRose.Behaviours.Objects.Bags.InventorySimpleBagRenderingManagementStrategy.SimpleBagInventorySubRenderer.Refresh ( )
virtual

Refreshes the content being rendered. This involves clearing everything, rendering each stack, and then applying a final rendering. These two steps are abstract and must be implemented by subclasses (since it is just a matter of the particular UI to create for them).

In these listings, position will match the slot because you are rendering everything (so the match will be automatic here).

Slot and key will differ - exactly by the amount of offset. So we try getting an element by a calculated key here by adding slot to offset.

int WindRose.Behaviours.Objects.Bags.InventorySimpleBagRenderingManagementStrategy.SimpleBagInventorySubRenderer.SlotFor ( int  position)
protected

Returns the slot index to use for certain visible position (according to paging settings).

Calculates the slot to match the position, according to current page and size. Returns -1 if that slot is not visible for the current page/size. Otherwise, returns a number between 0 and (PageSize - 1), or returns the input position as the slot if PageSize = 0;

Returns
If the given position is not meant to be visible, it returns -1. Otherwise, it returns the index between 0 and PageSize -1, or the same position if PageSize is 0

Returns the underlying simple bag (which is tied to the related renderer).

Paging will imply two properties: PageSize and Page. Both properties are unsigned integers:

  • Page size may be a positive value, or 0. If 0, no pagination will be applied: all the elements will be rendered. Otherwise, only a subset of N elements will be displayed at once.
  • Page makes sense when Page size is >= 0. Otherwise, page is 0. The page value will determine an offset of: N*PageSize elements.
  • There is a protected method if you want to change the paging later.
