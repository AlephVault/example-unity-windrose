Detailed Description

Aside of the map, Positionable objects are the spirit of the party.

Positionables are the middle step between the user interface (or artificial intelligence) and the underlying map and object strategies: They will provide the behaviour to move, teleport, attach to -and detach from- maps, and look in different directions.

They will also provide events to help other (dependent) behaviours to refresh appropriately (e.g. animation change, movement start, ...).

◆ Initialize()

void WindRose.Behaviours.Objects.Positionable.Initialize ( )

This method is called when the map is initialized (first) and when this object starts its execution in the scene. Both conditions have to be fulfilled for the logic to initialize.

For this method to succeed, this object must be a child object of one holding a ObjectsLayer which in turn must be inside a Map, and the map must have dimensions that allow this object considering its size and initial position.
